NOTE: The cluster-wide `enterprise.evalWorker` Deployment has been removed.
Eval-workers are now operator-managed, one per service group
(arena-eval-worker-<group>), and deploy automatically when a non-PromptKit,
eval-enabled AgentRuntime exists in a workspace. No `enterprise.evalWorker.*`
config is needed; the worker image is set via
`workspaceServices.evalWorker.image`.
{{- end }}

{{- if .Values.internalServiceAuth.enabled }}

NOTE: Internal service auth is ON (internalServiceAuth.enabled=true). session-api
requires a ServiceAccount bearer token; the operator wires this for every caller
it manages (facade, memory-api, eval-worker) plus the dashboard SA. If you run a
custom caller of session-api that cannot present an SA token, allowlist its
ServiceAccount or set `internalServiceAuth.enabled: false`. See the "Internal
service authentication" section of the chart README.
{{- else }}

WARNING: Internal service auth is OFF (internalServiceAuth.enabled=false).
session-api serves session CRUD without authentication to any pod that can reach
its ClusterIP. Set `internalServiceAuth.enabled: true` (the default) to close
SEC-1/SEC-5.
{{- end }}

To verify the operator is running:

  kubectl get pods -n {{ .Release.Namespace }} -l app.kubernetes.io/name={{ include "omnia.name" . }}

To deploy your first agent, create an AgentRuntime resource:

  kubectl apply -f - <<EOF
  apiVersion: omnia.altairalabs.ai/v1alpha1
  kind: AgentRuntime
  metadata:
    name: my-agent
  spec:
    promptPackRef:
      name: my-promptpack
    facades:
      - type: websocket
        port: 8080
    # Optional: customize the pod (workload identity, CSI secrets, GPU, etc.).
    # Full reference: https://omnia.altairalabs.ai/docs/how-to/configure-pod-overrides
    # podOverrides:
    #   serviceAccountName: my-workload-identity-sa
    #   nodeSelector:
    #     nvidia.com/gpu.product: A100
  EOF

=== Enterprise Features ===

Enterprise features are enabled including Arena for prompt testing and evaluation.

To create your first Arena evaluation:

1. Define a source for your PromptKit bundles:

  kubectl apply -f - <<EOF
  apiVersion: omnia.altairalabs.ai/v1alpha1
  kind: ArenaSource
  metadata:
    name: my-prompts
    namespace: <workspace-namespace>
  spec:
    type: git
    git:
      url: https://github.com/your-org/prompts
      ref:
        branch: main
  EOF

2. Run an evaluation job:

  kubectl apply -f - <<EOF
  apiVersion: omnia.altairalabs.ai/v1alpha1
  kind: ArenaJob
  metadata:
    name: my-eval
    namespace: <workspace-namespace>
  spec:
    sourceRef:
      name: my-prompts
    providerRef:
      name: my-provider
    scenarios:
      - name: test-scenario
  EOF

=== Admission Webhooks ===

⚠ Admission webhooks are ENABLED (webhook.enabled=true).
  This requires cert-manager in the cluster — it issues the serving cert
  ({{ .Values.webhook.certSecretName }}) and injects the webhook caBundle.
  Install it first, e.g.:
    helm repo add jetstack https://charts.jetstack.io
    helm install cert-manager jetstack/cert-manager \
      --namespace cert-manager --create-namespace --set crds.enabled=true
  On GKE Autopilot, also set cert-manager's leader election off kube-system
  (--set global.leaderElection.namespace=cert-manager).
  With Istio STRICT mTLS, exclude the webhook port on the operator pod:
    traffic.sidecar.istio.io/excludeInboundPorts: "{{ .Values.webhook.port }}"
{{- if .Values.enterprise.enabled }}

  Enterprise is enabled: the arena-controller also serves admission
  webhooks (ArenaSource/ArenaJob/ArenaTemplateSource). cert-manager issues
  its serving cert ({{ .Values.enterprise.arena.controller.webhook.certSecretName }})
  from the same self-signed Issuer. SessionPrivacyPolicy is validated by
  the operator. With Istio STRICT mTLS, also exclude the webhook port on
  the arena-controller pod:
    traffic.sidecar.istio.io/excludeInboundPorts: "{{ .Values.webhook.port }}"
{{- end }}
